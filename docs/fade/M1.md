# Fade M1 — Production Hardening

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** Shared CI setup

---

## Goal

Fade works for basic conversion but lacks the reliability features needed for daily use. Users need to be able to cancel jobs, see what went wrong, and inspect files before converting.

---

## Tasks

### Cancel / Pause Jobs

- [ ] Cancel button on each queued item during conversion
- [ ] Cancel sends SIGTERM to the FFmpeg/ImageMagick subprocess
- [ ] Cancelled items show "Cancelled" status (not "Error")
- [ ] Cancel All button to stop the entire batch
- [ ] Partial output files are cleaned up on cancel
- [ ] Pause/Resume for batch queue (pause between items, not mid-FFmpeg)

### Error Logging & Reporting

- [ ] Capture stderr from FFmpeg/ImageMagick subprocesses (currently discarded via `Stdio::null()`)
- [ ] On failure, show the actual error message in the UI (not just "conversion failed")
- [ ] Error details expandable in the queue item (click to see full stderr output)
- [ ] Log file: write conversion history to `~/.config/librewin/fade.log` (last 100 operations)
- [ ] Missing tool detection: on app start, check if `ffmpeg`, `ffprobe`, `magick` are in PATH — show install guidance if missing

### File Info Dialog

- [ ] "File Info" button or context menu option for queued files
- [ ] Shows: format, codec, resolution, duration, bitrate, file size, audio tracks
- [ ] Uses `ffprobe` for video/audio, `magick identify` for images
- [ ] Display estimated output size based on current conversion settings (rough estimate)

### Input Validation

- [ ] Validate trim_end > trim_start (currently no check)
- [ ] Validate crop dimensions fit within image bounds
- [ ] Validate custom resolution format before passing to FFmpeg
- [ ] Show validation errors inline (red border + message on the offending field)

### Tests

- [ ] Rust: test command building for image/video/audio conversions (verify FFmpeg args are correct)
- [ ] Rust: test output path construction with suffix
- [ ] Rust: test preset save/load/delete
- [ ] Frontend: test queue add/remove/clear
- [ ] Frontend: test tab auto-switching on file add
- [ ] Integration: test actual image conversion (JPEG → PNG) with sample file

---

## Definition of Done

- Users can cancel any running or queued conversion
- Failed conversions show the actual FFmpeg error message
- File info is inspectable before conversion
- Invalid settings are caught before conversion starts
- Missing external tools are detected and reported on startup
- Test suite covers command building and queue management
