# Stack M1 — Fix Broken Panels

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** Shared CI setup

## Goal

The right-side panel has UI that doesn't work. Search inputs aren't wired to CodeMirror. Format panel selectors are display-only. Fix everything that's built but broken.

---

## Tasks

### Search Panel (Wire to CodeMirror)

- [ ] Connect `searchQuery` state to CodeMirror's `@codemirror/search` API
- [ ] Find button triggers CM search with current query (not just opens CM's popup)
- [ ] Replace and Replace All work through the custom panel UI
- [ ] Case-sensitive toggle actually affects search behavior
- [ ] Match count displayed (e.g., "3 of 12 matches")
- [ ] Enter key in search field triggers find-next
- [ ] Escape closes search and returns focus to editor

### Format Panel (Make Functional)

- [ ] Line ending selector (LF/CRLF/CR) actually changes the document's line endings on save
- [ ] Encoding selector changes the encoding used when reading/writing files
- [ ] Tab size selector changes CodeMirror's tab width
- [ ] Indent style toggle (tabs vs spaces) affects editor behavior
- [ ] Changes persist per-tab (each open file remembers its format settings)

### Code Quality

- [ ] Add vitest config to Stack
- [ ] Smoke test: open file, edit, save, verify content
- [ ] Smoke test: search finds expected matches
- [ ] Smoke test: export HTML contains expected content

## Definition of Done

- Every UI control in the right panel does what it looks like it does
- Search/Replace works without opening CodeMirror's built-in search popup
- Format changes affect file output
- Basic test suite passes
