# Stack M2 — Markdown Experience

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** M1 (fix broken panels)

---

## Goal

Stack should feel like a markdown-native editor, not a plain text editor that happens to export markdown. Syntax should be visible as you type, and common formatting should be a shortcut away.

---

## Tasks

### Syntax Highlighting in Editor

- [ ] Bold text (`**text**`) renders with visual weight (bold font or distinct color)
- [ ] Italic text (`*text*`) renders in italic
- [ ] Headings (`# H1` through `###### H6`) render with size hierarchy
- [ ] Code spans (`` `code` ``) render with monospace background
- [ ] Code blocks (``` ``` ```) render with background and syntax color
- [ ] Links render with accent color, URL de-emphasized
- [ ] Lists (ordered + unordered) render with proper indentation markers
- [ ] Blockquotes render with left border and muted color
- [ ] Strikethrough (`~~text~~`) renders with line-through
- [ ] Use CodeMirror's `@codemirror/language` highlight tree — don't build custom regex parsing

### Markdown Shortcuts

- [ ] `Ctrl+B` → wrap selection in `**bold**`
- [ ] `Ctrl+I` → wrap selection in `*italic*`
- [ ] `Ctrl+K` → wrap selection in `[text](url)` link template
- [ ] `Ctrl+Shift+K` → insert code block
- [ ] `Ctrl+1` through `Ctrl+6` → set heading level
- [ ] `Ctrl+Shift+L` → toggle unordered list
- [ ] `Ctrl+Shift+O` → toggle ordered list
- [ ] `Ctrl+Shift+Q` → toggle blockquote
- [ ] `Ctrl+Shift+X` → toggle strikethrough
- [ ] All shortcuts work as toggles (apply if not present, remove if already applied)

### Toolbar (Optional)

- [ ] Thin formatting toolbar below tabs (bold, italic, heading, list, code, link)
- [ ] Toolbar can be hidden via preference
- [ ] Toolbar buttons reflect current cursor position state (bold button highlighted when cursor is in bold text)

### Tests

- [ ] Test markdown shortcut toggles (apply + remove)
- [ ] Test syntax highlighting renders expected classes
- [ ] Test that shortcuts work with and without selection

---

## Definition of Done

- Markdown syntax is visually distinct as you type (headings are bigger, bold is bold, code has background)
- All standard markdown formatting has a keyboard shortcut
- Shortcuts toggle correctly (don't double-wrap)
