# Prism M1 — Metadata & UX Polish

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** Shared CI setup

---

## Goal

Prism's core viewing works well. This milestone adds the features users expect from a media viewer: seeing file details, opening files easily, and knowing what shortcuts are available.

---

## Tasks

### Metadata Viewer

- [ ] Info panel (toggleable sidebar or overlay) showing file metadata
- [ ] **Images:** Resolution, color space, bit depth, file size, EXIF data (camera, lens, date taken, GPS if present)
- [ ] **Video:** Resolution, codec, bitrate, duration, frame rate, audio tracks
- [ ] **Audio:** Codec, bitrate, sample rate, duration, channels, ID3 tags (artist, album, title)
- [ ] **PDF:** Page count, author, title, creation date, PDF version
- [ ] **3D Models:** Vertex count, face count, bounding box dimensions, material count
- [ ] Rust backend: new command `get_metadata(path)` using ffprobe for media, exiftool or `kamadak-exif` crate for images, pdf.js metadata API for PDFs
- [ ] Toggle with keyboard shortcut (I key)
- [ ] Copy metadata values to clipboard on click

### Drag-and-Drop

- [ ] Drag files onto Prism window to open them
- [ ] Drag from Shelf (via file path) opens in Prism
- [ ] Drop zone visual feedback (overlay with "Drop to open")
- [ ] Support dropping multiple files → opens first, queues rest (for playlist feature later)

### Keyboard Shortcut Overlay

- [ ] Press `?` to show all available keyboard shortcuts
- [ ] Overlay shows shortcuts grouped by viewer type (image, video, audio, PDF, 3D)
- [ ] Escape dismisses overlay
- [ ] Shortcuts listed in the overlay are the actual bound shortcuts (not hardcoded docs)

### File Navigation

- [ ] Open file dialog (Ctrl+O) using rfd
- [ ] Recent files list (stored in `~/.config/librewin/prism-recent.json`, max 20)
- [ ] Next/Previous file in same directory (Left/Right arrows when not in video/audio seek mode)
- [ ] File name and path shown in titlebar or status bar

### Tests

- [ ] Rust: test metadata extraction for sample image, video, audio, PDF files
- [ ] Frontend: test keyboard shortcut overlay renders all shortcuts
- [ ] Test drag-and-drop file opening

---

## Definition of Done

- User can see detailed metadata for any opened file
- Files can be dragged onto the window to open them
- `?` shows all keyboard shortcuts
- Ctrl+O opens a file dialog
- Arrow keys navigate through files in the same directory
