# Splice M2 — Preview & Parallel Encoding

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** M1 (production hardening)

---

## Goal

Users should be able to see what their output will look like before committing to a full conversion, and batch jobs should run faster.

---

## Tasks

### Preview

- [ ] **Image preview:** Show before/after side-by-side for image operations (resize, crop, format change)
- [ ] After preview generates a low-res version using the current settings
- [ ] Preview updates live as settings change (debounced ~300ms)
- [ ] Crop preview: visual overlay on the original image showing the crop region
- [ ] Drag to adjust crop region

- [ ] **Video preview:** Thumbnail strip showing frames at intervals
- [ ] Trim preview: show first/last frame of the trimmed region
- [ ] No full video preview (too expensive) — just frame thumbnails

- [ ] **Audio preview:** Waveform visualization of the audio file
- [ ] Trim markers shown on waveform (drag to adjust)
- [ ] Play 3-second preview of trim start/end points

### Parallel Encoding

- [ ] Configurable concurrency: 1, 2, or 4 simultaneous conversions
- [ ] Default: 2 (balance between speed and system responsiveness)
- [ ] Queue processing respects concurrency limit
- [ ] Individual progress bars still work per-item
- [ ] Overall progress accounts for all running items
- [ ] CPU usage indicator (optional — show system load)

### Persistent Queue

- [ ] Save queue state to `~/.config/librewin/splice-queue.json` on app close
- [ ] Restore queue on app reopen (items marked "pending" only — completed/error items are cleared)
- [ ] Option to clear saved queue on restore ("Resume previous session?" prompt)

### Tests

- [ ] Test image preview generation matches expected output
- [ ] Test parallel queue processing completes all items
- [ ] Test queue persistence save/load cycle

---

## Definition of Done

- Image operations show live before/after preview
- Video trim shows frame thumbnails at start/end points
- Audio shows waveform with draggable trim markers
- Batch processing runs 2+ conversions simultaneously
- Queue survives app restart
