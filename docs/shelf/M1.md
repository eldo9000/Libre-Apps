# Shelf M1 — Archive Manager Integration

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Complete
**Depends on:** Shared CI setup

---

## Goal

Archive browsing and extraction is no longer a separate app — it's a built-in Shelf feature. Users should be able to browse into archives like folders and extract files without leaving the file manager.

---

## Tasks

### Rust Backend

- [x] Add `zip` crate (or `zip2`) for ZIP read/write support
- [x] Add `sevenz-rust` or shell out to `7z` for 7Z support
- [x] Add `tar` + `flate2` crates for TAR/GZ support
- [x] New command: `list_archive(path)` → returns Vec<ArchiveEntry> with name, size, modified date, is_dir
- [x] New command: `extract_archive(path, dest)` → extracts full archive to destination
- [x] New command: `extract_files(path, files, dest)` → extracts selected files only
- [x] New command: `archive_info(path)` → returns format, total size, file count, compressed size
- [x] Handle nested archives (archive within archive) — display, don't auto-extract

### Frontend (QuickFiles.svelte)

- [x] Detect archive files by extension (.zip, .7z, .tar, .tar.gz, .tar.bz2, .tar.xz, .rar)
- [x] Double-click/Enter on archive → browse into it (call `list_archive` instead of `list_dir`)
- [x] Breadcrumb trail shows archive path distinctly (e.g., icon or color change for archive segment)
- [x] "Extract Here" and "Extract To..." in context menu for archives
- [x] "Extract Selected" when inside an archive with files selected
- [x] Back button exits archive and returns to parent directory
- [x] Archive files show compressed/uncompressed size in file list

### Edge Cases

- [x] Password-protected ZIPs — prompt for password, display error if wrong
- [x] Corrupted archives — show error toast, don't crash
- [x] Very large archives (10,000+ files) — paginate or lazy-load the listing
- [x] Symlinks inside archives — display but warn about extraction outside target

### Tests

- [x] Rust unit tests for archive listing (ZIP, 7Z, TAR)
- [x] Rust unit tests for extraction (full + selective)
- [x] Test corrupted archive handling
- [x] Test nested archive detection

---

## Definition of Done

- User can double-click a ZIP/7Z/TAR file and browse its contents like a folder
- User can extract full archive or selected files via context menu
- Archives are visually distinct in the breadcrumb trail
- Corrupted/password-protected archives show appropriate errors
- Rust tests pass for all archive operations
