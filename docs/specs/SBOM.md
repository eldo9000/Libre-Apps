# Software Bill of Materials (SBOM)

Every Freedom App release includes CycloneDX SBOMs for both its Rust and npm dependency trees.
SBOMs are generated by the CI `generate-sbom` workflow job and uploaded as signed release artifacts.

## Artifacts per app

| File | Contents | Tool |
|------|----------|------|
| `{app}.rust.sbom.json` | Rust transitive dependencies (from `Cargo.lock`) | `cargo-sbom` |
| `{app}.rust.sbom.json.minisig` | Signature | minisign |
| `{app}.npm.sbom.json` | npm transitive dependencies (from `package-lock.json`) | `npm sbom` |
| `{app}.npm.sbom.json.minisig` | Signature | minisign |

Format: [CycloneDX JSON](https://cyclonedx.org/specification/overview/) — version 1.4 for Rust, 1.5 for npm.

## Verifying an SBOM

```bash
# Download SBOM and signature
curl -LO https://github.com/eldo9000/Libre-Apps/releases/latest/download/shelf.rust.sbom.json
curl -LO https://github.com/eldo9000/Libre-Apps/releases/latest/download/shelf.rust.sbom.json.minisig

# Verify signature
minisign -Vm shelf.rust.sbom.json -P RWSvBX8a5MExBbOsCmBqDLxNBb8ofBef1k3eqI79Z/LSGp/DBj1YwW5S
```

## Inspecting an SBOM

Any CycloneDX-compatible tool can consume these files.

```bash
# List all components and their versions (requires jq)
jq '.components[] | "\(.name) \(.version)"' shelf.rust.sbom.json

# Check for a specific package
jq '.components[] | select(.name == "openssl")' shelf.rust.sbom.json

# Count total dependencies
jq '.components | length' shelf.rust.sbom.json
```

Online viewers: [CycloneDX BOM validator](https://cyclonedx.org/tool-center/), Dependency-Track.

## LibreWin-OS integration

The LibreWin install manager downloads and verifies SBOMs at install time. Current use:

1. **Signature verification** — same minisign key as binaries. Install aborts if signature fails.
2. **Audit hook** — the install manager can optionally run `osv-scanner` or `grype` against
   the SBOM before completing installation, flagging known CVEs to the user.

The SBOM is not required to be clean for installation to proceed — that is a user-configurable
policy in the install manager. The default is: warn on high/critical CVEs, block on critical
CVEs with available fixes.

## Tooling

| Tool | Version | Source |
|------|---------|--------|
| `cargo-sbom` | latest stable | `cargo binstall cargo-sbom` |
| `npm sbom` | built-in | npm 10+ / Node 22 |

`cargo-sbom` reads the Cargo workspace at `apps/Cargo.toml` and generates a CycloneDX JSON
document listing all transitive Rust dependencies with their versions, crate registry URLs,
and checksums.

`npm sbom --package-lock-only` reads `package-lock.json` without requiring `node_modules`,
making it fast and side-effect-free in CI.
