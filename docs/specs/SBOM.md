# Software Bill of Materials (SBOM)

Every Freedom App release includes CycloneDX SBOMs for both its Rust and npm dependency trees.
SBOMs are generated by the CI `generate-sbom` workflow job and uploaded as signed release artifacts.

## Artifacts per app

| File | Contents | Tool |
|------|----------|------|
| `{app}.sbom.json` | Combined Rust + npm dependencies (from `Cargo.lock` + `package-lock.json`) | `syft` |
| `{app}.sbom.json.minisig` | Signature | minisign |

Format: [CycloneDX JSON](https://cyclonedx.org/specification/overview/). `syft` detects all package ecosystems present in the app directory and merges them into a single document.

## Verifying an SBOM

```bash
# Download SBOM and signature
curl -LO https://github.com/eldo9000/Libre-Apps/releases/latest/download/shelf.sbom.json
curl -LO https://github.com/eldo9000/Libre-Apps/releases/latest/download/shelf.sbom.json.minisig

# Verify signature
minisign -Vm shelf.sbom.json -P RWSvBX8a5MExBbOsCmBqDLxNBb8ofBef1k3eqI79Z/LSGp/DBj1YwW5S
```

## Inspecting an SBOM

Any CycloneDX-compatible tool can consume these files.

```bash
# List all components and their versions (requires jq)
jq '.components[] | "\(.name) \(.version)"' shelf.sbom.json

# Check for a specific package
jq '.components[] | select(.name == "openssl")' shelf.sbom.json

# Count total dependencies
jq '.components | length' shelf.sbom.json
```

Online viewers: [CycloneDX BOM validator](https://cyclonedx.org/tool-center/), Dependency-Track.

## LibreWin-OS integration

The LibreWin install manager downloads and verifies SBOMs at install time. Current use:

1. **Signature verification** — same minisign key as binaries. Install aborts if signature fails.
2. **Audit hook** — the install manager can optionally run `osv-scanner` or `grype` against
   the SBOM before completing installation, flagging known CVEs to the user.

The SBOM is not required to be clean for installation to proceed — that is a user-configurable
policy in the install manager. The default is: warn on high/critical CVEs, block on critical
CVEs with available fixes.

## Tooling

| Tool | Version | Source |
|------|---------|--------|
| `syft` | latest stable | `anchore/sbom-action/download-syft` GitHub Action |

`syft` scans the app directory and detects all package ecosystems present — Cargo.lock (Rust)
and package-lock.json (npm) — combining them into a single CycloneDX JSON SBOM. It is
installed via the official `anchore/sbom-action/download-syft` GitHub Action which pins to
a specific digest and verifies the binary before use.
