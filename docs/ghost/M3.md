# Ghost M3 — Security Audit & Hardening

**Phase:** 2 (Power Features & Integration)
**Status:** Not started
**Depends on:** M2 (settings & browser fundamentals)

---

## Goal

Ghost handles untrusted web content. Before shipping, it needs a thorough security review and hardening pass. Ghost has no pro tier — it stays free and minimal.

---

## Tasks

### Content Security Policy

- [ ] Implement strict CSP in tauri.conf.json
- [ ] Block inline script execution from web content reaching Tauri APIs
- [ ] Ensure toolbar injection script is allowlisted but arbitrary page scripts cannot invoke Tauri commands
- [ ] Test: malicious page cannot call `invoke('close_window')` or any Tauri command

### Toolbar Injection Audit

- [ ] Review the 4.9KB embedded JavaScript toolbar for XSS vectors
- [ ] Ensure URL bar sanitization blocks all dangerous protocols (javascript:, data:, vbscript:, blob:)
- [ ] Ensure toolbar DOM cannot be manipulated by page JavaScript (use Shadow DOM or isolated world)
- [ ] Test: page JS cannot read or modify toolbar state
- [ ] Test: page JS cannot intercept toolbar navigation events

### WebView Isolation

- [ ] Verify each tab's webview is process-isolated (if WebKitGTK supports it)
- [ ] Ensure one tab crashing doesn't bring down the app
- [ ] Block navigation to `file://` and `tauri://` URLs from web content
- [ ] Block popups unless user-initiated (click-triggered)

### Download Safety

- [ ] Scan download filename for dangerous extensions (.desktop, .sh, .exe, .bat, .cmd)
- [ ] Warn before downloading executable files
- [ ] Block downloads to locations outside ~/Downloads
- [ ] Sanitize filenames (remove path traversal attempts like `../`)

### Privacy Verification

- [ ] Automated test: verify no data persists in WebKit cache dirs after close
- [ ] Automated test: verify no cookies survive app restart
- [ ] Automated test: verify WebRTC is disabled (use WebRTC leak test page)
- [ ] Automated test: verify third-party requests are blocked for known trackers

### Permissions

- [ ] Block geolocation requests by default
- [ ] Block notification permission requests by default
- [ ] Block camera/microphone access by default
- [ ] Allow clipboard read/write (needed for normal browsing)
- [ ] Show permission request UI for any future permission requests

### Tests

- [ ] Security: XSS injection test against toolbar
- [ ] Security: Tauri command invocation from untrusted page
- [ ] Security: file:// URL navigation from web content
- [ ] Security: download path traversal
- [ ] Privacy: post-session data verification

---

## Definition of Done

- Independent security review of toolbar injection
- CSP blocks unauthorized Tauri API access
- No data persists after app close (verified by automated tests)
- Dangerous downloads are warned/blocked
- Web content cannot access system resources
- All permission requests blocked by default
