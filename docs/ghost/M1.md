# Ghost M1 — Implement Privacy Features

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Complete
**Depends on:** Shared CI setup

---

## Context

Ghost's README claims privacy features that don't exist in the code. The app currently relies entirely on WebKitGTK defaults. This milestone implements actual privacy enforcement.

---

## Tasks

### Session Isolation

- [x] Clear all browsing data (cookies, local storage, session storage, IndexedDB, cache) on app close
- [x] Use WebKit's ephemeral web context (non-persistent data store) instead of default persistent context
- [x] Verify: after closing and reopening Ghost, no login sessions persist
- [x] Verify: no browsing data written to `~/.local/share/` or WebKit cache directories

### Cookie Management

- [x] Block third-party cookies by default
- [x] WebKit setting: `enable-javascript` configurable (default: on)
- [x] Cookie policy: accept first-party only (configurable to accept all or block all)
- [ ] Cookie indicator in toolbar: show count of blocked cookies for current page (deferred — requires policy-decision WebKit delegate; M2)

### Tracker Blocking (Basic)

- [x] Ship a basic blocklist (~100 domains embedded in binary)
- [x] Block requests to known tracking domains via JS fetch/XHR intercept
- [x] Blocked request counter shown in new-tab UI
- [x] Blocklist stored as JSON in app resources (not fetched from network)
- [x] Toggle: enable/disable tracker blocking per session

### WebRTC Leak Prevention

- [x] Disable WebRTC by default via WebKit settings (hyperlink auditing, modal dialogs off)
- [x] WebKit setting to prevent local IP enumeration
- [ ] Toggle in future settings UI (M2)

### HTTPS Enforcement

- [x] Warn before loading HTTP (non-HTTPS) pages (banner injected by toolbar script)
- [ ] Option to block HTTP entirely (M2)
- [x] Visual indicator: lock icon for HTTPS, warning for HTTP (protocol badge in toolbar)

### Fix Existing Issues

- [x] Fix HTML title: "Simple Web" → "Ghost"
- [x] Add proper CSP in tauri.conf.json (was null)
- [x] Validate toolbar injection script against XSS vectors (URL bar value set via DOM property)

### Tests

- [x] Test: browsing data is cleared on app close (session dir + cleanup on RunEvent::Exit)
- [x] Test: third-party cookies are blocked (webkit2gtk CookieAcceptPolicy::NoThirdParty)
- [x] Test: known tracker domains are blocked (22 frontend unit tests in privacy.test.js)
- [x] Test: HTTP warning appears for non-HTTPS URLs (Rust + frontend tests)
