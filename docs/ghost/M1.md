# Ghost M1 — Implement Privacy Features

**Phase:** 1 (Foundation & MVP Completion)
**Status:** Not started
**Depends on:** Shared CI setup

---

## Context

Ghost's README claims privacy features that don't exist in the code. The app currently relies entirely on WebKitGTK defaults. This milestone implements actual privacy enforcement.

---

## Tasks

### Session Isolation

- [ ] Clear all browsing data (cookies, local storage, session storage, IndexedDB, cache) on app close
- [ ] Use WebKit's ephemeral web context (non-persistent data store) instead of default persistent context
- [ ] Verify: after closing and reopening Ghost, no login sessions persist
- [ ] Verify: no browsing data written to `~/.local/share/` or WebKit cache directories

### Cookie Management

- [ ] Block third-party cookies by default
- [ ] WebKit setting: `enable-javascript` configurable (default: on)
- [ ] Cookie policy: accept first-party only (configurable to accept all or block all)
- [ ] Cookie indicator in toolbar: show count of blocked cookies for current page

### Tracker Blocking (Basic)

- [ ] Ship a basic blocklist (EasyList-derived, ~5000 domains)
- [ ] Block requests to known tracking domains via WebKit resource load delegate
- [ ] Blocked request counter shown in toolbar
- [ ] Blocklist stored as JSON in app resources (not fetched from network)
- [ ] Toggle: enable/disable tracker blocking per session

### WebRTC Leak Prevention

- [ ] Disable WebRTC by default (prevents IP leak behind VPN)
- [ ] WebKit setting to prevent local IP enumeration
- [ ] Toggle in future settings UI

### HTTPS Enforcement

- [ ] Warn before loading HTTP (non-HTTPS) pages
- [ ] Option to block HTTP entirely
- [ ] Visual indicator: lock icon for HTTPS, warning for HTTP

### Fix Existing Issues

- [ ] Fix HTML title: "Simple Web" → "Ghost"
- [ ] Add proper CSP in tauri.conf.json (currently null)
- [ ] Validate toolbar injection script against XSS vectors

### Tests

- [ ] Test: browsing data is cleared on app close
- [ ] Test: third-party cookies are blocked
- [ ] Test: known tracker domains are blocked
- [ ] Test: HTTP warning appears for non-HTTPS URLs

---

## Definition of Done

- Ghost actually enforces privacy (not just claims it)
- Cookies from previous sessions don't persist
- Third-party cookies blocked by default
- Known trackers blocked with visible counter
- WebRTC disabled by default
- HTTP pages show security warning
- CSP is properly configured
